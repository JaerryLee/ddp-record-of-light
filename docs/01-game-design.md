# 01 · Game Design

> 본 문서는 **메커닉의 명세**를 담는다. 서사·레벨·비주얼은 각각 [02](02-narrative.md), [03](03-level-design.md), [04](04-art-direction.md)로 분리한다.

## 1. High-Level Structure

```
Boot  →  Main Menu  →  Intro (45s) → Zone 1 → Zone 2 → Zone 3 → Ending (3m) → Credits
                                    └──── Hub transition (코리더) ────┘
```

- 총 플레이타임: **15~22분** (숙련 플레이어 12분, 막힌 플레이어 25분 상한)
- 세이브: **씬 전환 시 자동**. 수동 세이브 없음
- 체크포인트: 각 Zone 진입 시

## 2. Core Loop

### 2.1 30-초 루프 (Minute-to-Minute)
```
1. 빛 관찰  (광원·반사 지점 파악)
     ↓
2. 프리즘 배치  (Node에 스냅 + 각도 선택)
     ↓
3. 빔 확인  (실시간 시각화, 실패 시 1로)
     ↓
4. 크리스털 활성화  (빔이 크리스털에 도달하면 자동)
     ↓
5. 짧은 홀로그램·보이스 재생  (15~25초)
     ↓
6. 다음 퍼즐 또는 존 출구 해금
```

### 2.2 존 단위 루프 (Zone Arc)
1. **진입** — 환경 톤과 광원 소개 (BGM 테마 전환)
2. **학습** — 새 메커닉 (존당 1개) 튜토리얼 퍼즐
3. **심화** — 학습된 메커닉을 응용한 퍼즐 1~2개
4. **보상** — 크리스털 활성화 + 시네마틱 보이스
5. **전환** — 다음 존으로 이어지는 내러티브 코리더

## 3. Controls (PC Reference Build)

### 3.1 입력 매핑

| 행동 | 기본 키 | 비고 |
|---|---|---|
| 전후좌우 이동 | `W A S D` | 이동 속도 3.5 m/s (걷기) |
| 시선 | `Mouse Look` | 감도 기본 `1.0x`, 0.2 ~ 3.0 조절 |
| 점프 | `Space` | 높이 1.2 m. 퍼즐엔 사용 ❌, 탐험용만 |
| 상호작용 (E) | `E` | Node 선택·크리스털 활성화·사인 읽기 |
| 프리즘 소환/해제 | `좌클릭` | 조준한 Node에 토글 |
| 프리즘 회전 | `Q / E` (Node 소유 중) | 15° 단위 스냅 |
| 프리즘 색 스위치 | `1 / 2 / 3` | Zone 2 이후: White/R/G/B |
| 수집 아카이브 | `Tab` | 이미 얻은 크리스털 회상 메뉴 |
| 일시정지 | `Esc` | 설정·타이틀 이동·저장 확인 |
| 스크린샷(포토모드) | `F12` | HUD 토글 |

### 3.2 입력 시스템
- **Unity New Input System** 기반. `Player.inputactions`에 전 매핑 선언
- Gamepad 지원: Xbox/PS 매핑 제공(본선 진출 시). 1차 제출은 키보드·마우스 전용

### 3.3 접근성 (1차 범위)
| 항목 | 상태 |
|---|---|
| 자막 on/off + 크기 3단계 | ✅ |
| 모션 블러 · 헤드밥 비활성화 | ✅ |
| 색약 대응 (색 기반 퍼즐) | ✅ 각 색에 고유 패턴 오버레이 |
| FOV 슬라이더 60~90° | ✅ |
| 키 리바인딩 | ⏳ 본선 확장 |

## 4. Prism System (핵심 메커닉)

### 4.1 용어 정의
- **Node** — 월드에 배치된 고정 슬롯. 벽·바닥·공중 장치 어디든 디자인 가능
- **Prism** — Node에 붙는 빛 굴절체. 1종 (Zone 1 기본) → 3종(Split/Filter/Mirror, Zone 2 이후)
- **Beam** — 한 번에 그려지는 최대 3 회 반사, 40 m 거리의 라인 광선

### 4.2 프리즘 유형
| 종류 | 해금 | 기능 | 시각 |
|---|---|---|---|
| **Mirror** | Zone 1 | 빔을 한 방향으로 반사 (스냅 90°/45°) | 은빛 정육각형 |
| **Splitter** | Zone 2 | 입사 백색광을 R/G/B 3빔으로 분할 | 삼각 프리즘 |
| **Filter** | Zone 2 | 특정 색만 통과 (나머지는 차단) | 색유리 디스크 |
| **Mirror+** | Zone 3 | 광원의 시간차(사이클)에 연동되는 반사체 | 회전 금속 블레이드 |

### 4.3 배치 규칙
- Node에 **snap-only**. 자유 배치 불가 → 레벨 디자인 의도 보존
- 한 Node엔 프리즘 1개
- 존별 동시 배치 가능 수 상한: Zone1 `1`, Zone2 `3`, Zone3 `4`
- 배치·해제에 쿨다운 없음. 즉시 반영

### 4.4 빔 규칙
- 빔은 **시각적으로 항상 가시** (플레이어가 광 경로를 손쉽게 읽어야 함)
- 반사 횟수 ≤ **3** (과도한 퍼즐 난이도 상한)
- 빔과 크리스털이 **1초 이상 지속 접촉** 시 활성화 (깜빡임에 오발동 방지)
- 빔이 플레이어 몸체를 통과해도 데미지 없음 (전투 없음)

### 4.5 퍼즐 설계 가이드라인 (디자이너용)
1. **하나의 교훈, 하나의 퍼즐** — 각 퍼즐은 정확히 하나의 메커닉 변주를 가르친다
2. **죽은 퍼즐 금지** — 실수로 끼임(soft-lock) 상태 없게 프리즘 해제는 항상 가능
3. **시각 언어 우선** — 텍스트 힌트 금지. 공간·조명·색으로만 유도
4. **해답 1개 원칙** — 여러 풀이가 있을 때는 정해진 '의도 해답'만 크리스털을 활성화 (서사적 정합성)

## 5. Memory Crystal System

### 5.1 활성화 조건
| 크리스털 | 필요 색 | 필요 프리즘 | 위치 |
|---|---|---|---|
| C1 · "개관 2014" | White | Mirror ×1 | Zone 1 진입홀 |
| C2 · "첫 밤, 첫 불" | White | Mirror ×2 | Zone 1 외벽 |
| C3 · "런웨이" | Red | Splitter + Filter(R) | Zone 2 전시홀 |
| C4 · "큐레이터의 밤" | Blue | Splitter + Filter(B) | Zone 2 아카이브 |
| C5 · "옥상의 두 사람" | Green | Splitter + Filter(G) | Zone 3 잔디 언덕 |
| C6 · "대정전의 밤" | White | Mirror+ ×2 (시간제) | Zone 3 엔딩 지점 |

### 5.2 재생 Vignette
- 지속: **15~25초** (C6만 45초, 엔딩 프리앰블)
- 요소: 홀로그램 실루엣(반투명) + VARCO Voice + 서브타이틀 + 앰비언스 덕킹
- 플레이어 이동은 가능. 카메라 강제 고정 없음 (몰입 유지 ≠ 통제)
- 스킵: 3초 이상 `Esc` 홀드

### 5.3 수집 메뉴 (Archive)
- `Tab`으로 열람. 이미 활성화한 크리스털 다시 듣기 가능
- UI는 DDP 내벽에 프로젝션되는 디제틱 패널로 표현 (풀스크린 모달 아님)

## 6. Progression & Pacing

### 6.1 비트 차트 (목표)
| 시각 | 이벤트 | 플레이어 감정 |
|---|---|---|
| 0:00 | 인트로 페이드 인, 아카이브 AI의 첫 멘트 | 호기심 |
| 0:45 | 첫 조작 가능, 빔 가시화 | 발견 |
| 3:00 | C1 활성화, 첫 보이스 | 충족 |
| 6:00 | Zone 2 진입, Splitter 해금 | 도전 |
| 10:00 | C3 또는 C4 | 집중 |
| 13:00 | Zone 3 진입, 옥상 개방 | 경외 |
| 17:00 | C5 | 서정 |
| 19:00 | C6 + 엔딩 시퀀스 | 여운 |
| 22:00 | 크레딧 | 잔향 |

### 6.2 난이도 곡선
- 퍼즐 난이도: **1 → 2 → 3 → 3 → 3 → 2**  (마지막 C6는 시간 압박으로 체감 3, 해답은 2)
- 의도적으로 Zone 3 C6를 쉽게 두어 **엔딩의 감정**이 난이도에 가리지 않게 함

### 6.3 힌트 시스템
- 자동 힌트 없음. 단, **3분 이상 퍼즐 진전 없음** 감지 시 아카이브 AI가 1회 모호한 단서 음성을 재생
- 옵션에서 힌트 보이스 off 가능

## 7. Economy & Collectibles

본작은 기본적으로 **경제 시스템 없음**. 수집 요소는 단일:

| 수집물 | 수량 | 보상 |
|---|---|---|
| 메모리 크리스털 | 6 | 각 보이스 씬 + 엔딩 해금 |
| 히든 "도시의 틈" | 3 (본선 확장에서) | 제작자 코멘터리 |

1차 제출 빌드는 히든 요소 없음 — 완성도 집중을 위해.

## 8. Failure States

- **사망 없음**. 낙사 위험 지역은 난간/역장으로 물리적으로 차단
- 퍼즐 실패 = 그냥 다시 풀면 됨. 페널티 없음
- **유일한 종료 경로는 엔딩**. 플레이어가 끝낼 수 없는 상태는 디자인 오류

## 9. Systems Summary (Scripts 관점)

자세한 구현 구조는 [07 · Technical Design](07-technical-design.md) 참조. 여기엔 요약만.

- `PlayerController` — 1인칭 이동 + 인터랙션 레이캐스트
- `PrismSystem` — Prism/Node 상태·배치·회전
- `BeamSystem` — 광선 경로·반사·색 마스크 계산, LineRenderer 시각화
- `CrystalSystem` — 활성화 조건 평가, 비넷 트리거
- `VignettePlayer` — 홀로그램 + 보이스 + 자막 동기화
- `ArchiveAI` — 상태 기반 힌트·내레이션 타이밍
- `ZoneController` — 씬 진입/퇴장 이벤트, 게이팅
